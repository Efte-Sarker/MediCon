This `docs/` directory is the single source of truth for the MediCon project.

Before performing any task, read the documentation in the following order:

1. 01-product-spec.md (Application features and requirements.)
2. 02-architecture.md (Project structure and architecture.)
3. 03-coding-standards.md (Coding conventions and implementation standards.)
4. 04-safety-and-compliance.md (Medical, security, privacy, and emergency safety rules.)
5. 05-workflow.md (Development workflow and Definition of Done.)
6. 06-development-roadmap.md (Implementation milestones and build order.)

# Conflict Resolution
The project documentation is designed to be consistent. If two documents appear to provide conflicting guidance, do not merge or ignore the instructions.

Apply them in the following priority order:

1. Safety & Compliance (04-safety-and-compliance.md)
2. Architecture (02-architecture.md)
3. Coding Standards (03-coding-standards.md)
4. Workflow (05-workflow.md)
5. Product Specification (01-product-spec.md)

The Product Specification defines feature scope and intended behavior but does not override higher-priority technical or safety requirements.

If a conflict cannot be resolved using this priority, stop and ask for clarification instead of making assumptions.
